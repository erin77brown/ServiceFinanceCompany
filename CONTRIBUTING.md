# Contribution Guidelines

Thank you for your interest in contributing to svcfin com!  
We welcome developers of all skill levels to improve the project.

---

## How to Contribute

1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your feature or fix.
4. Follow coding standards and best practices.
5. Commit changes with clear messages.
6. Push changes to your fork.
7. Open a Pull Request to the main branch.

---

## Reporting Issues

If you find a bug:
- Check if it already exists in Issues.
- Provide detailed steps to reproduce.
- Include screenshots if applicable.
- Mention your environment (OS, Node version, browser).

---

## Suggesting Features

We welcome feature suggestions:
- Clearly explain the feature.
- Describe its benefit.
- Provide mockups or examples if possible.

---

## Code of Conduct

- Be respectful and inclusive.
- Provide constructive feedback.
- Maintain professionalism in discussions.
- No harassment, discrimination, or inappropriate behavior.

---

## Review Process

- All pull requests will be reviewed.
- Changes may be requested before approval.
- Once approved, it will be merged into the main branch.
- Contributors will be credited for accepted changes.
